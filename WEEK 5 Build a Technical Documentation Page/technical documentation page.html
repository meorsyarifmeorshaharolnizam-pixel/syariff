Introduction

Python is a high-level, interpreted, object-oriented programming language with dynamic semantics. Its high-level built in data structures, combined with dynamic typing and dynamic binding, make it very attractive for Rapid Application Development.

Python's simple, easy to learn syntax emphasizes readability and therefore reduces the cost of program maintenance. It supports modules and packages, which encourages program modularity and code reuse.

To print a simple greeting sequence on your terminal interface, execute this standard string:

print("Hello, World!")
Variables and Types

Python is completely object oriented, and not "statically typed". You do not need to declare variables before using them, or declare their type. Every variable in Python is an object.

Numbers and strings can easily be assigned and evaluated concurrently using operations:

my_integer = 42 my_float = 3.14

You can verify object classes explicitly using internal properties.

Control Flow

Python uses the usual flow control statements known from other languages, with some modifications. The if statement is used for conditional execution flows based on evaluated truths.

There can be zero or more elif parts, and the else part is completely optional. Here is an example of an evaluation construct block:

if x < 0: print('Negative')
Defining Functions

The keyword def introduces a function definition statement block. It must be followed by the function name and the parenthesized list of formal parameters.

The statements that form the body of the function start at the next line, and must be indented carefully to comply with standard compilation parameters.

Key architectural properties of functions include:

  • Keyword arguments are matched by position or name parameter keys.
  • Default argument values can be pre-assigned during initial functional definitions.
  • Functions always return a value, defaulting to None if left blank.
Working with Lists

Python knows a number of compound data types, used to group together other values. The most versatile is the list, which can be written as a list of comma-separated values between square brackets.

Lists might contain items of different types, but usually the items all have the same type structural definition traits.

Common utility mutations possible within index positions:

  • Appending values directly to the rear end of lists.
  • Slicing structural fragments using boundary offsets.
